# باتش دعم اللغة العربية (RTL) لتطبيق ChatGPT Desktop

تعديل (Patch) برمجي جاهز لتطبيق **ChatGPT Desktop** الرسمي، يحل مشكلة اتجاه النص (RTL)، ويحسن توافق الكتابة باللغتين العربية والعبرية، والنصوص المدمجة (عربي/إنجليزي)، مع ضبط محاذاة علامات الترقيم، والحفاظ على الأكواد البرمجية من اليسار لليمين (LTR).

بواسطة **يسري حادي (Yosri Hadi)** - [yosrihadi/ChatGpt-Rtl](https://github.com/yosrihadi/ChatGpt-Rtl)

---

## لمن هذا المشروع؟

هذا المشروع موجه للمستخدمين العرب (وكذلك مستخدمي العبرية) الذين يعتمدون على تطبيق OpenAI ChatGPT Desktop ويرغبون في الحصول على تجربة كتابة طبيعية من اليمين لليسار داخل التطبيق، وذلك دون المساس بملفات التثبيت الأصلية للتطبيق.

## التثبيت - بخطوة واحدة (سطر أوامر)

### Windows

قم بفتح **PowerShell** (لا يشترط وجود صلاحيات المسؤول Admin)، انسخ السطر التالي، ثم اضغط Enter:

```powershell
irm https://raw.githubusercontent.com/yosrihadi/ChatGpt-Rtl/main/install-online.ps1 | iex

```

هذا كل شيء! في النهاية سيظهر لك اختصار جديد باسم **"ChatGPT"** على سطح المكتب وفي قائمة إبدأ (Start Menu)، والذي سيقوم بفتح النسخة المعدلة الداعمة للغة العربية.

> **المتطلبات:** تثبيت [Node.js (LTS)](https://nodejs.org/) + تطبيق ChatGPT الرسمي من متجر Microsoft Store.
> لا يتطلب التثبيت صلاحيات مسؤول (Admin) أو تعديل أذونات النظام.

> **ملاحظة حول إنذارات الأمان (Windows Defender):**
> إذا أظهر لك الويندوز تحذيراً أمنياً (مثل `Trojan:Win32/ClickFix`)، فهذا **إنذار كاذב (False Positive)**. نظام الويندوز يقوم بتعليم أي أمر من نوع `irm ... | iex` كإجراء مشبوه بسبب **طريقة التنفيذ عبر الإنترنت**، وليس بسبب محتوى الكود نفسه. السكريبت مفتوح المصدر بالكامل هنا في GitHub؛ وهو يقوم فقط بإنشاء **نسخة محلية** من تطبيق ChatGPT لتعديل الواجهة دون العبث بملفات النظام (Registry/Services). إذا ظهر التحذير: اذهب إلى **Windows Security → Protection History → اختر العنصر → "Allow" (السماح)**، ثم أعد تشغيل الأمر.

### macOS (تجريبي)

دعم نظام الماك متاح ولكنه لا يزال في الطور التجريبي. يتبع السكريبت الأسلوب القياسي لتعديل تطبيقات Electron على أجهزة الماك (تخطي حماية `codesign` وحماية ASAR) ويستخدم نفس كود الجافاسكريبت المستخدم في ويندوز.

افتح **Terminal** والصق الأمر التالي:

```bash
curl -fsSL https://raw.githubusercontent.com/yosrihadi/ChatGpt-Rtl/main/install-online.sh | bash

```

سيقوم هذا بإنشاء تطبيق معدل في المسار `~/Applications/ChatGPT-RT-AI.app` مع دعم العربية، دون المساس بالتطبيق الأصلي `ChatGPT.app` الموجود في مجلد التطبيقات الأساسي.

> **المتطلبات:** تثبيت [Node.js](https://nodejs.org/) (عبر `brew install node`) + أدوات مطوري آبل (`xcode-select --install`) + تطبيق ChatGPT الرسمي.

## قبل / بعد التعديل

**ما الذي يتغير فعلياً:**

* **قبل الباتش:** النصوص العربية تظهر ملتصقة باليسار، علامات الاستفهام والترقيم تظهر معكوسة، والأسطر التي تدمج بين العربية والإنجليزية (مثل الأكواد) تبدو متداخلة وغير مقروءة.
* **بعد الباتش:** المحادثات العربية تُحاذى لليمين بشكل مثالي، علامات الترقيم في مكانها الطبيعي، والأكواد البرمجية تحافظ على اتجاهها من اليسار لليمين لكي لا تتشوه.

**ما الذي يكتشفه الباتش تلقائياً:**

* ✅ **العربية/العبرية في مربع الكتابة:** يتم ضبط اتجاه النص (RTL) فوراً أثناء الكتابة.
* ✅ **الردود المباشرة (Streaming):** يتم ضبط محاذاة كل فقرة بشكل مستقل بناءً على لغتها.
* ✅ **النصوص المدمجة:** يتم التعرف عليها استناداً لأول حرف قوي (first-strong detection).
* ✅ **صناديق الأكواد (Code Blocks):** تحافظ **دائماً** على اتجاهها LTR (سواء كانت Monaco أو Markdown).
* ✅ **الأكواد المضمنة (Inline code):** تبقى LTR حتى وإن كانت داخل فقرة عربية.
* ✅ **علامات الترقيم الطائرة:** يتم تثبيتها عبر خاصية `unicode-bidi: plaintext`.

**Windows:**

```powershell

```

**macOS:**


عملية الإزالة تحذف النسخة المعدلة فقط، وتطبيق ChatGPT الأصلي **لا يتأثر أبداً** ويستمر بالعمل كالمعتاد.

## تحديثات تطبيق ChatGPT

عندما تقوم شركة OpenAI بتحديث تطبيق ChatGPT عبر متجر مايكروسوفت، **لن يتم تحديث نسختك المعدلة تلقائياً**. للحصول على التحديث الجديد مع دعم العربية، قم ببساطة بتشغيل أمر التثبيت مرة أخرى (السطر المذكور بالأعلى)، وسيقوم السكريبت تلقائياً باكتشاف النسخة الجديدة وتعديلها.

## كيف يعمل الباتش داخلياً؟

1. يبحث عن التطبيق الأصلي داخل مسار الويندوز المخفي `C:\Program Files\WindowsApps\OpenAI.Codex_...\app`.
2. ينسخ الملفات بالكامل إلى مجلد المستخدم `%LOCALAPPDATA%\Programs\ChatGPT-RT-AI`.
3. يفك حزمة الواجهة `resources\app.asar` باستخدام `@electron/asar`.
4. يدمج كود الجافاسكريبت `codex-rtl-payload.js` داخل ملفات الواجهة الأساسية (مثل `composer-*.js` و `index-*.js`).
5. يعيد تجميع حزمة `app.asar`.
6. يعطل فحص السلامة `EnableEmbeddedAsarIntegrityValidation` في ملف `Codex.exe` باستخدام `@electron/fuses` للسماح بتشغيل الحزمة المعدلة.
7. يكتب ملف تحقق (`resources\rt-ai-codex-rtl-patch.json`).
8. ينشئ اختصاراً جديداً `ChatGPT.lnk` على سطح المكتب وفي قائمة إبدأ.

العملية برمتها تتم داخل مسار `%LOCALAPPDATA%` الخاص بالمستخدم. لا يوجد أي مساس بملفات `WindowsApps` الأساسية، ولا بملفات الـ Registry، ولا بخدمات الويندوز.

## قيود معروفة (Known limitations)

* **تحديثات التطبيق:** تتم عبر المتجر وتستهدف النسخة الأصلية، لذا يجب إعادة تشغيل السكريبت بعد كل تحديث رسمي.
* **نسخة الماك:** لا تزال تجريبية ومبنية على أساسيات حقن تطبيقات Electron.
* **النسخة المعدلة غير موقعة رسمياً:** في ويندوز تفقد النسخة المعدلة توقيع MSIX الرقمي، وفي الماك تأخذ توقيعاً محلياً (Ad-hoc).
* **نموذج الثقة:** السكريبت يتم تنفيذه عبر سطر الأوامر مباشرة. المستودع صغير ومفتوح المصدر، يرجى قراءة الأكواد قبل التنفيذ.
